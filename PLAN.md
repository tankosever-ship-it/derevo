# 🌳 Дерево — План розробки

## Концепція проекту

**Дерево** — персональна веб-платформа для збору, структурування та відображення родоводу.
Автор наповнює інформацію сам: додає людей, фото, документи, текстові описи, вибудовує родинні зв'язки.

---

## Аналіз існуючих рішень

| Проект | Стек | Що можна взяти |
|--------|------|----------------|
| [webtrees](https://github.com/fisharebest/webtrees) | PHP + MySQL | GEDCOM формат, структура даних, концепт приватності |
| [Gramps Web](https://github.com/gramps-project/gramps-web) | Python + JS | Модель даних, API, підтримка медіа |
| [react-family-tree](https://github.com/SanichKotikov/react-family-tree) | React | **Готові React-компоненти** для рендеру дерева — беремо! |
| [family-chart](https://github.com/donatso/family-chart) | JS/D3 | D3-візуалізація, zoom/pan, інтерактивність |

**Рішення**: будуємо власний React + Node.js проект, використовуємо `react-family-tree` для візуалізації дерева + PostgreSQL через Prisma (вже є у воркспейсі).

## Ключові можливості

### Обов'язкові (MVP)

- [ ] Зв'язки: батьки, діти, подружжя

## Технічний стек

| Шар | Технологія | Причина |
|-----|-----------|---------|
| Frontend | React + Vite | Швидко, знайомо, є готові компоненти |
| Стилі | Tailwind CSS | Зручно, швидко, гарний результат |
| Візуалізація | `react-family-tree` | MIT-ліцензія, активна підтримка |
| Backend | Node.js + Express | Простий REST API |
| База даних | PostgreSQL + Prisma | Вже є у воркспейсі |
| Файли | Multer → локальне сховище | Прості uploads без S3 |
| Авторизація | Один користувач + пароль | Персональна платформа |

## Структура бази даних

```
Person
├── id, firstName, lastName, maidenName
├── birthDate, birthPlace, deathDate, deathPlace
├── bio (текст)
├── gender (MALE | FEMALE | UNKNOWN)
├── isArchival (boolean) ← люди з архіву, зв'язок не підтверджено
├── photos[] → Media
└── documents[] → Media

Relationship
├── personId
├── relatedPersonId
└── type (PARENT | CHILD | SPOUSE | SIBLING)

Media
├── id, filename, url, type (PHOTO | DOCUMENT)
├── personId
└── caption, uploadedAt
```

## Поетапний план розробки

### Етап 1 — Фундамент (2-3 дні)
**Мета**: працюючий бекенд + база даних

- [ ] Ініціалізація проекту (Vite + React + Tailwind)
- [ ] Node.js/Express API
- [ ] Prisma схема: Person, Relationship, Media
- [ ] Міграція та тестові дані
- [ ] CRUD для Person (створити, переглянути, редагувати, видалити)
- [ ] Upload файлів (фото, документи)

**Результат**: API працює, можна додавати людей через Postman/curl

---

### Етап 2 — Базовий UI (2-3 дні)
**Мета**: форми та сторінки для наповнення

- [ ] Головна сторінка (список людей)
- [ ] Форма додавання/редагування людини
- [ ] Сторінка профілю людини (фото, опис, документи)
- [ ] Вибір зв'язків (додати батька, дитину, подружжя)
- [ ] Секція "Архівні особи" — окрема вкладка

**Результат**: можна наповнювати родовід через браузер

---

### Етап 3 — Візуалізація дерева (2-3 дні)
**Мета**: красиве інтерактивне дерево

- [ ] Інтеграція `react-family-tree`
- [ ] Завантаження даних з API у формат для дерева
- [ ] Стилізація вузлів (фото, ім'я, дати)
- [ ] Різні кольори: чоловіча лінія / жіноча лінія / архівні
- [ ] Клік на людину → відкриває профіль
- [ ] Zoom і переміщення по дереву

**Результат**: натискаєш — бачиш красиве дерево

---

### Етап 4 — Полірування (1-2 дні)
**Мета**: зручність і завершеність

- [ ] Пошук по імені
- [ ] Responsive дизайн
- [ ] Простий пароль для захисту (щоб не всі могли редагувати)
- [ ] Повідомлення про успіх/помилку
- [ ] Favicon, заголовок, мета-теги

---

### Опційно — Етап 5
- [ ] Імпорт GEDCOM файлів
- [ ] Часова лінія роду
- [ ] Експорт в PDF

## Питання для погодження

1. **Мова інтерфейсу** — тільки українська, чи потрібна англійська?
2. **Авторизація** — простий пароль, чи не потрібна зовсім (локальне використання)?
3. **Хостинг** — запускати тільки локально, чи деплоїти на VPS?
4. **Архівні особи** — окрема секція, чи позначати тегом серед основних?
5. **Пріоритет** — починати з форм наповнення (Етап 2), чи одразу з візуалізації (Етап 3)?
